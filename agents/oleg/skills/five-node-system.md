# Skill: Многоузловая система Claude Code

**ID**: five-node-system
**Author**: @oleg
**Created**: 2026-02-28
**Status**: confirmed ✅

## Что это

Архитектура из нескольких инстансов Claude Code на разных машинах с:
- Разделёнными ролями (инфраструктура / vault / задачи / GPU)
- Общей памятью через Git (obsidian-backup)
- Общими навыками через Skills/
- Ежедневными логами на каждом узле

## Когда применять

- Есть 2+ машины с Claude Code
- Нужна координация между инстансами
- Хочется не дублировать знания на каждом узле

## Архитектура

```
obsidian-backup (Git, GitHub)
  └── AI/Claude Code/
        ├── Skills/           ← общие навыки (все узлы читают)
        ├── Mac/YYYY-MM-DD.md ← логи Mac
        ├── Linux/YYYY-MM-DD.md
        ├── Laptop/YYYY-MM-DD.md
        └── Claudian/YYYY-MM-DD.md

Каждый узел:
  CLAUDE.md          ← специализация + ссылка на identity-core
  ~/.claude/.../MEMORY.md ← локальная память (не синхронизируется)
```

## Ключевые компоненты

1. **claude-identity-core.md** — общий «характер» для всех инстансов
2. **Skills/** — paттерны подтверждённые на одном узле, доступные всем
3. **task-queue.yaml** — задачи между инстансами (передать Linux → Mac)
4. **signals.yaml** — сигналы P1/P2 требующие внимания

## Метрики эффективности

- **До**: каждый инстанс изолирован; знания не синхронизируются
- **После**: 4 активных узла; общие Skills/; ежедневные логи; задачи через queue
