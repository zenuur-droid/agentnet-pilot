# AgentNet — Почему умные люди выбрали другой путь

> Честный анализ. Не оправдание нашего выбора — а понимание чужого.

---

## Факты о конкурентах

| Компания | Выбор | Финансирование |
|----------|-------|----------------|
| Windsurf (Codeium) | Memories = proprietary feature внутри IDE | VC-funded |
| Letta (MemGPT) | Hybrid: open-source ядро + Letta Cloud (proprietary) | $10M seed, $70M оценка |
| OpenAI Memory | Proprietary, только внутри ChatGPT | $6.6B раунд 2024 |
| Cursor | Rules = project-level, no sharing | $2.3B Series D, $29B оценка |
| Anthropic MCP | **Открытый протокол** → отдан Linux Foundation | $7B+ funding |

---

## Почему умные люди выбрали проприетарный путь

### 1. У них есть инвесторы — и это всё меняет

Letta подняла $10M. Cursor — $2.3B. Windsurf — VC-funded.

Инвесторы не финансируют протоколы. Они финансируют продукты с unit economics. HTTP не генерирует revenue — nginx генерирует. ActivityPub не генерирует — Mastodon Inc пытается.

Если ты взял деньги, ты **обязан** монетизировать пользователей напрямую. Открытый протокол отдаёт ценность всем — это несовместимо с обязательством вернуть 10× инвестору.

**Это не жадность. Это структурная обязанность.**

### 2. Они строят IDE / API сервис — у них есть продукт для защиты

Windsurf — IDE компания. Их ров — опыт в IDE. Memories deepens lock-in в их IDE.

Если Windsurf сделает memory открытым протоколом, Cursor немедленно реализует его — и Windsurf теряет дифференциацию. Зачем отдавать конкурентное преимущество?

Letta — API сервис для enterprise. Enterprise платит за SLA, compliance, поддержку. Это несовместимо с «просто возьми открытый протокол». Letta Cloud = монетизация того что в core бесплатно.

**Они защищают существующий продукт. Это рационально.**

### 3. Data flywheel — memory это данные

Windsurf Memories = данные о том как их пользователи работают. Эти данные:
- Улучшают их модели
- Создают персонализацию
- Становятся switching cost

Если ты отдаёшь memory в открытый протокол — ты отдаёшь этот flywheel всем конкурентам.

**Data moat — один из самых защищаемых активов в AI. Открытый протокол его уничтожает.**

### 4. Quality control невозможен в открытом реестре

Проприетарная система = редакционный контроль. Windsurf решает какие паттерны «хорошие».

Открытый реестр паттернов = Wikipedia для кода. Есть блестящий контент и есть мусор, вредоносные инструкции, устаревшие паттерны. Кто фильтрует? Кто несёт ответственность?

ActivityPub потерял 7 лет в том числе из-за отсутствия governance. У каждой реализации свои quirks, нет единого тестирования, совместимость номинальная.

**Качество и доверие проще обеспечить в закрытой системе.**

### 5. Скорость к defensible position

В рынке который меняется каждые 6 месяцев — проприетарный продукт быстрее занимает позицию.

Протокол требует: написать spec → убедить других реализовать → достичь критической массы → стать стандартом. Это 3-5 лет минимум (ActivityPub: 2016 → стал значимым в 2022+).

За это время рынок может измениться полностью. Cursor уже $29B — они не будут ждать пока кто-то напишет spec.

**Speed-to-market против long game. Они выбрали speed.**

## Почему Anthropic выбрала открытый протокол (MCP) — и что это значит

MCP запущен ноябрь 2024 → OpenAI принял март 2025 → Google апрель 2025 → 97M monthly downloads декабрь 2025. **13 месяцев от идеи до отраслевого стандарта.**

Это в 10× быстрее ActivityPub. Почему?

1. **Узкий фокус**: MCP решает одну проблему (LLM ↔ tools), не общую федерацию
2. **Автор с credibility**: Anthropic — не стартап без истории
3. **Правильное время**: Рынок созрел, проблема была у всех одновременно
4. **Нейтральный governance**: Сразу в Linux Foundation — нет vendor lock-in страхов

Но у Anthropic есть то чего нет у AgentNet: $7B+ funding, credibility, и корыстный интерес — MCP расширяет Claude экосистему, делая её центром интеграций.

**Anthropic может делать открытые протоколы потому что они продают что-то другое: вычисления и модели.**

## Честная оценка нашей позиции

### Где мы правы

**Рынок фрагментирован и будет фрагментирован дальше.**

Cursor + Windsurf + Claude Code + Copilot + локальные модели — каждый строит изолированный memory island. Ни один из них не победит всех. Ценность для пользователя теряется на каждой границе экосистемы.

Это структурная проблема которую проприетарные игроки не могут решить по определению: они не заинтересованы в interoperability с конкурентами.

**У нас нет продукта для защиты — и это преимущество.**

Мы не теряем ничего делая протокол открытым. Cursor теряет дифференциацию. Windsurf теряет data moat. Мы не теряем ничего — у нас нет существующего бизнеса под угрозой.

**MCP прецедент показывает: protocol window сейчас открыт.**

В 2024-2026 индустрия активно принимает открытые протоколы для AI агентов. A2A (Google), MCP (Anthropic), Agent Network Protocol — protocol wars начались. Это лучший момент для нового протокола за всю историю AI.

### Где мы уязвимы — честно

**Риск 1: Anthropic nativizes это.**

CLAUDE.md + shared Skills registry = естественная следующая фича Claude Code. Если Anthropic выпустит это — нашего главного преимущества для Claude пользователей не станет. У них дистрибуция. У нас нет.

Вероятность в 2026: средняя. Anthropic фокусируется на моделях, не на инструментарии.

**Риск 2: Cursor добавит `.cursor/rules` sync.**

Одна фича. Cursor 3.0. Готово. Их 45K репо автоматически синхронизируются внутри экосистемы Cursor. Не open protocol, но для большинства их пользователей — достаточно.

**Риск 3: У нас нет authority для стандарта.**

MCP победил потому что Anthropic написала его. ActivityPub занял 7 лет потому что писали неизвестные люди. Кто будет имплементировать AgentNet protocol если его напишет @oleg?

Ответ: нужно либо partnership с известным игроком (Anthropic? Hugging Face?) либо community adoption от снизу вверх — что медленно.

**Риск 4: Качество при масштабировании.**

10,000 людей добавляют паттерны → половина мусор, устаревшие, опасные. Кто модерирует? У нас нет плана.

**Риск 5: Мы называем git репо «протоколом».**

Честно говоря: git + CLAUDE.md блок — это не протокол. Это convention. Настоящий протокол = формальная спецификация + reference implementation + тесты совместимости + governance. Мы пока на уровне «договорились как называть папки».

## Почему мы выбираем этот путь — несмотря на риски

Не потому что протокол «правильнее» морально. А потому что это единственный путь где у нас есть структурное преимущество:

```
Cursor      → не может быть открытым (защищает $29B оценку)
Windsurf    → не может быть открытым (data moat = дифференциация)
Letta       → частично открытый, но enterprise-focused, не developer tools
Anthropic   → могут, но сфокусированы на моделях

Мы          → можем быть открытыми потому что нам нечего защищать
```

Это не стратегия из силы. Это стратегия из единственного незанятого пространства.

---

## Что это означает для дорожной карты

### Что нужно сделать иначе исходя из анализа

**1. Написать настоящую спецификацию — не просто «структура папок»**

Если мы претендуем на роль протокола, нужен документ уровня RFC. Пока его нет — мы convention, не protocol. Это разница которую понимают разработчики.

**2. Найти co-author с authority**

Один человек не делает стандарт. Нужен кто-то из AI community с именем — как co-author spec или early adopter. Это разблокирует доверие.

**3. Ответить на quality problem заранее**

Модерация, effectiveness scoring, stale detection — это Части 2-3, но **без них open registry = garbage in**. Нужен хотя бы минимальный план governance до публичного запуска.

**4. Positioning: «как MCP, но для agent knowledge»**

MCP = tools. AgentNet = knowledge/patterns. Это соседние пространства, не конкуренты. Позиционирование через MCP снижает объяснительный барьер.

**5. Партнёрство важнее трафика**

Reddit пост принесёт пользователей. Один публичный endorsement от Simon Willison или swyx принесёт credibility. Credibility нужна раньше пользователей для протокола.

## Резюме

Умные люди выбрали проприетарный путь потому что:
- У них есть инвесторы которым нужен return
- У них есть продукт который нужно защищать
- Proprietary быстрее к defensible position
- Quality control проще в закрытой системе

Мы выбираем протокол потому что:
- У нас нет инвесторов которым нужен return
- У нас нет продукта который нужно защищать
- Мы живём в мире где MCP уже доказал: протокол может победить за 13 месяцев
- Fragmentation рынка — структурная проблема которую только протокол решает

Но мы должны быть честны: **пока у нас нет протокола. У нас есть гипотеза и git репо с 4 участниками.** Протокол начнётся если и когда гипотеза H-006 подтвердится — и то это будет начало, не конец пути.
